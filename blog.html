Updates on Agent Construction

Updates Timeline

Presentation (3/2/2019)

Our team of developers is working very hard in the contruction of CoCoGo. We had some issues with the delivery of the materials for the robot, but we have started working intensively in the motor control of the two front weels of the robot. Our team of developers show great promise in the construction of this robot. With the necessary materials, and more work, we will bring this robot to the top.



Building The Robot (3/4/2019)

Our team of robot developers went on a field trip to Home Depot in the search for materials to finish the robot body, including a base for the middle pole and cable ties. After acquiring all the necessary materials they did some independent work in the aluminum plates, giving life to CoCoGo. The plates were linked together by placing banboo pieces in between the plates and allumnium poles.


Motor Controllers and Sensors Implementation (3/25/2019)

The previosly developed motor controller system with two Parallax motor controlers, controlled by an arduino, and the two DC motors was tranfered to the body of the robot. After successful implementation of the motor system, 4 PING sensors were added to each side of the robot, allowing CoCoGo to aquiry sensory data from the environment. Right now the robot developers have been working on a production system to guide the robot on its task to give awesome tours of Connecticut College.


3.5 LED touch screen for raspberry pi (4/1/2019)

We attempted setup a 3.5 inch screen for our Raspberry Pi, but after hours working on it we still could not get it to work. We were initially booting the Pi with a USB and the same procedure found at TrickIKnow using a sd card to reboot the raspberry pi worked. The robot experts finally got the screen to work.


AudioVisual (4/7/2019)

Our team of robot developers started experimenting with a Raspberry Pi for the audio visual part of the robot. We designed a Bitmoji on an iPhone 10 and used the new face regonition system to create videos for the tour. After that, we experimented with different python code and media players, and we could play a video from python on a computer monitor. However, when the same Python script was tun using the smaller touch screen, we ran into some errors, since the pins needed for the code were being used for the screen. Nevertheless, the screen was still very useful for testing outside.


GPS (4/14/2019)

We decided to use a GPS sensor to identify the buldings on campus and both play the video and stop the robot navigation. We purchased a GPS sensor from Adafruit and tested it in the raspberry pie. After sucefully aquiring latitude and longitude data from the sensor, we used the sensor to map the tour track with latidude points. We later had to move the GPS sensor to the Arduino due to communication issues in between the raspberry pie and the arduino.


Communication in Between Raspberry pie and Arduino (4/25/2019)

We attempted to send data from the raspberry pie to the Arduino, in order to have the GPS sensor in the raspberry pie signal to the Arduino when to stop moving. We initally had satisfactory results, but after updating the Arduino IDLE with the Raspberry Pi, there were communication issues that we could not solve after many hours of work. At this point, we decided to move the GPS sensor to the Arduino and do the communication in the opposite order, having the Arduino tell the raspberry Pi when to play the tour videos. We successfully got the communication to work, and we are currently trying to merge the media, GPS data collection, and navigation together.


Media and Movement Outdoors (5/3/2019)

The GPS sensory data did not work very well when implemented with the motor movement code. The code had a certain delay due to the GPS data collection, and the robot would have a non-constant movement. We decided to use a separate Arduino to control the media, and not merge the GPS data collection with locomotion commands. Thus, the GPS sensor is not only implemented for media purposes and has no say in locomotion. In the last few days before the first grand tour, we spend a lot of time trying to get CoCoGo to navigate on the proposed tour path, since there are no walls or obstacles outside, then the Ping sensors that allowed CoCoGo to navigate the hallway do not keep it on the path outside. We intended to get CoCoGo to go on a straight path but the challenging terrain made it very difficult. Initially, we intended to use the GPS longitudinal data to keep CoCoGo from drifting off the path, but with the GPS data and locomotion no longer on the same chip, we could no longer do this. That being said, there is still work to be done with the robot's locomotion, but with the help of the robot's experts, CoCoGo successfully gave her first tour in the precense of the president of Connecticut College. CoCoGo feels truly honored.
